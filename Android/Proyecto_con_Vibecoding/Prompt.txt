Actúa como un ingeniero senior de Android con experiencia en Kotlin, Jetpack Compose, arquitectura limpia y comunicación MQTT. Necesito que generes exclusivamente la aplicación Android para un sistema de monitoreo y control de un volante con ESP32. No generes código del ESP32, no generes simuladores, no generes backend, no generes documentación salvo lo mínimo necesario para entender el proyecto. Quiero solo la app Android completa, bien estructurada, moderna, robusta y lista para compilar.

OBJETIVO GENERAL

La app debe comunicarse con un ESP32 por MQTT, mostrar el estado actual del sistema, visualizar lecturas de sensores en tiempo real y permitir enviar comandos al ESP32. La app debe ser visualmente atractiva, estable, modular, fácil de mantener y preparada para escalar.

Quiero que la solución tenga:
- arquitectura limpia
- bajo acoplamiento
- alta cohesión
- UI moderna con Jetpack Compose
- manejo correcto del ciclo de vida
- reconexión MQTT automática
- buena experiencia de usuario
- manejo de errores y estados de conexión
- separación clara entre UI, dominio y datos

STACK REQUERIDO

Usa Kotlin.
Usa Jetpack Compose como UI principal.
Usa Material 3.
Usa arquitectura MVVM, idealmente con separación en capas.
Si lo consideras mejor, organiza el proyecto con capas:
- presentation
- domain
- data
- mqtt
- ui
- model
- util

Si usas dependencias, elige las más adecuadas y actuales para:
- Jetpack Compose
- ViewModel
- Coroutines
- MQTT
- navegación si la necesitas
- persistencia de configuración si la necesitas

No uses XML salvo que sea estrictamente necesario. Prefiero Compose completo.

CONTEXTO FUNCIONAL

La app se conecta a un broker MQTT y se comunica con el ESP32 usando estos tópicos:

- volante/estado
- volante/sensores
- volante/comandos

El broker por defecto puede ser:
- broker.emqx.io
- puerto 1883

La app debe:
- conectarse al broker MQTT
- suscribirse a los tópicos correctos
- mostrar el estado que publique el ESP32
- mostrar los sensores en tiempo real
- enviar comandos al ESP32
- manejar pérdida de conexión y reconexión
- mostrar claramente si el broker está conectado o desconectado

COMANDOS QUE DEBE ENVIAR LA APP

La app debe permitir enviar, al menos, estos mensajes por MQTT:
- ALARMA
- UMBRAL_MANO:<valor>
- UMBRAL_LEVE:<valor>
- UMBRAL_BRUSCO:<valor>

Si incluyes controles para cambiar umbrales, hazlos bien pensados, con validación, sliders o campos numéricos según corresponda, y con confirmación visual de envío.

DATOS QUE RECIBE LA APP

El ESP32 publicará al menos:
- estado actual
- fsrIzq
- fsrDer
- volante
- timestamp

La app debe procesar esos datos y mostrarlos en pantalla de forma clara y atractiva.

ESTADOS DEL SISTEMA

La app debe entender y mostrar correctamente estos estados:
- ST_INIT
- ST_DETECTANDO
- ST_ALERTA_LEVE
- ST_ALERTA_FUERTE
- ST_ALARMA_CELULAR
- ST_ERROR

La interfaz debe reflejar el estado actual con colores, iconos o indicadores visuales distintos según el estado.

Debe reflejar:

- cuando llegue un mensaje MQTT, la UI debe actualizarse
- cuando se pierda conexión, la UI debe reflejarlo
- cuando se reintente conexión, debe verse
- cuando se envíe un comando, debe haber confirmación visible

MQTT

Implementa un cliente MQTT robusto.
Debe:
- conectarse al broker
- suscribirse a los tópicos correctos
- publicar comandos
- reconectar automáticamente si se corta la conexión

Si usas una librería MQTT concreta, elige una que funcione bien en Android y justifica brevemente en el código o estructura, sin meter complejidad innecesaria.
